Planejamento de Sprints: Help! SENAI

Consta por ser um período curto e fixo em que uma equipe scrum trabalha para concluir uma quantidade definida de trabalho.
Neste modelo, toda sprint será de uma semana

Sprints - Primeiro mês

Sprint 1:

Duração:

1 semana (2 dias úteis)

Prioridade:

Login/Autenticação, Página Principal

Objetivo

Implementar a funcionalidade de login e a página principal.

Conteúdo da Sprint:

  • Desenvolver a interface de login e a integração com a API da instituição escolar.
  • Implementar o sistema de autenticação e armazenamento de usuários.
  • Criar a página principal com a exibição de mapas.
  • Desenvolver os testes unitários para o login e a página principal.

    • Sprint 2:

    Duração:

    1 semana (2 dias úteis)

    Prioridade:

    Eventos, Pesquisar Eventos

    Objetivo

    Implementar a funcionalidade de gerenciamento de eventos e pesquisa de eventos.

    Conteúdo da Sprint:

  • Criar as interfaces para a adição, edição e remoção de eventos.
  • Implementar a lógica de armazenamento e organização de eventos.
  • Desenvolver a funcionalidade de pesquisa de eventos por data, categoria, nome ou palavra-chave.

    • Sprint 3:

    Duração:

    1 semana (2 dias úteis)

    Prioridade:

    Chat

    Objetivo

    Implementar o sistema de chat em tempo real.

    Conteúdo da Sprint:

  • Integrar o sistema de chat com a interface do website.
  • Desenvolver a lógica de envio e recebimento de mensagens e arquivos.
  • Implementar o histórico de conversas.
  • Desenvolver os testes unitários para o sistema de chat.

    • Sprint 4:

    Duração:

    1 semana (2 dias úteis)

    Prioridade:

    Calendário de Datas

    Objetivo

    Implementar a funcionalidade de adicionar, editar e remover datas importantes no calendário.

    Conteúdo da Sprint:

  • Criar a interface para a gestão de datas no calendário.
  • Implementar a lógica de armazenamento e organização de datas.
  • Desenvolver os testes unitários para o gerenciamento de datas no calendário.

    • Sprints - Segundo mês

    Sprint 5:

    Duração:

    1 semana (2 dias úteis)

    Prioridade:

    Visualização do Calendário

    Objetivo

    Implementar as diferentes visualizações do calendário (mensal, semanal, diário).

    Conteúdo da Sprint:

  • Desenvolver as interfaces para as diferentes visualizações do calendário.
  • Implementar a lógica de navegação entre as visualizações.
  • Desenvolver os testes unitários para as diferentes visualizações do calendário.

    • Sprint 6:

    Duração:

    1 semana (2 dias úteis)

    Prioridade:

    Atualização de Informações

    Objetivo

    Implementar a funcionalidade para os usuários alterarem suas informações de registro.

    Conteúdo da Sprint:

  • Criar a interface para a atualização de informações.
  • Implementar a lógica de validação e atualização das informações do usuário.
  • Desenvolver os testes unitários para a atualização de informações.

    • Sprint 7:

    Duração:

    1 semana (2 dias úteis)

    Prioridade:

    Testes

    Objetivo

    Implementar os testes de funcionalidade e segurança.

    Conteúdo da Sprint:

  • Desenvolver testes de funcionalidade para todas as features do sistema.
  • Implementar testes de segurança para garantir a proteção dos dados do usuário.
  • Criar documentação de testes e resultados.

    • Sprint 8:

    Duração:

    1 semana (2 dias úteis)

    Prioridade:

    Documentação

    Objetivo

    Criar documentação detalhada do sistema.

    Conteúdo da Sprint:

  • Criar documentação para usuários e administradores.
  • Criar tutoriais e materiais de apoio.
  • Implementar a documentação online.

    • Sprints - Terceiro mês

    Sprint 9-12:

    Duração:

    1 semana (2 dias úteis) por sprint

    Prioridade:

    Melhorias e Novas Features

    Objetivo

    Implementar melhorias e novas funcionalidades de acordo com o feedback dos usuários e necessidades da instituição.

    Conteúdo da Sprint:

  • Implementar as melhorias e novas funcionalidades com base no feedback recebido.
  • Ajustar a interface e a experiência do usuário.
  • Realizar testes e garantir a qualidade do sistema.
