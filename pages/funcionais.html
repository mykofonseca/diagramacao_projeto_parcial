Requisitos Funcionais: Help! SENAI

Funcionalidades específicas que o sistema legado deve possuir

Requisitos Funcionais

Nº Requisito Descrição
1 O sistema deve permitir que o usuário efetue login com suas credenciais fornecidas pela instituição.
2 O sistema deve verificar as credenciais do usuário com o sistema de autenticação da instituição
3 O sistema deve armazenar as informações do usuário após o login bem-sucedido.
4 O sistema deve exibir uma página principal com uma amostra de mapas das áreas da instituição.
5 O sistema deve permitir que a instituição escolar adicione, edite e remova eventos.
6 O sistema deve exibir uma lista de eventos com datas, categorias e descrições.
7 O sistema deve fornecer uma interface de chat em tempo real entre usuários e a instituição escolar.
8 O sistema deve permitir que os usuários enviem mensagens e arquivos.
9 O sistema deve permitir que a instituição escolar adicione, edite e remova datas importantes no calendário.
10 O sistema deve permitir que os usuários visualizem as datas importantes.
11 O sistema deve permitir que os usuários alterem sua senha, email ou número de telefone.
12 O sistema deve solicitar a autenticação do usuário antes de permitir a modificação de informações.

Descrição dos requisitos funcionais

O sistema precisa garantir acesso seguro e personalizado para usuários autenticados pela instituição. Ele deve apresentar uma interface intuitiva que permita a navegação e localização dentro do campus, além de disponibilizar informações sobre eventos e datas importantes. A comunicação entre usuários e a instituição escolar deve ser eficiente, por meio de um sistema de chat que permita o envio de mensagens e arquivos. Por fim, o sistema deve permitir a atualização de dados pessoais dos usuários de forma segura, com autenticação antes de qualquer alteração.
